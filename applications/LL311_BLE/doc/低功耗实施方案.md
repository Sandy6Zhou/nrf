**版本**: V1.0
**作者**: Harrison Wu (wuyujiao@jimiiot.com)
**日期**: 2026.03.23

---

## 1. 方案概述

### 1.1 目标
实现全模块统一的低功耗管理，达到以下功耗指标：

| 状态 | 预期电流 |
|------|---------|
| 全速运行 | ~10-20mA |
| 智能模式 | ~3-5mA |
| 长续航模式 | ~100-500μA |
| 关机模式 | ~10-50μA |

### 1.2 设计原则

1. **统一接口**: 所有模块通过 `my_pm` 框架管理
2. **总线独立**: I2C21/I2C22/UART 各自独立管理
3. **引用计数**: 支持多设备共享总线
4. **状态机驱动**: 明确的生命周期管理
5. **失败回退**: 操作失败自动恢复状态

## 4. 关键实现

### 4.1 电源管理回调接口

```c
/* 设备操作回调结构体 */
struct PM_DEVICE_OPS
{
    int (*suspend)(void);   /* 模块进入低功耗 */
    int (*resume)(void);    /* 模块恢复正常工作 */
    int (*init)(void);      /* 模块初始化（首次） */
};
```

### 4.2 NFC 模块实现示例

```c
/* ========== 前置声明 ========== */
static int nfc_pm_init(void);
static int nfc_pm_suspend(void);
static int nfc_pm_resume(void);

/* ========== 回调结构体定义 ========== */
static const struct PM_DEVICE_OPS nfc_pm_ops = {
    .init = nfc_pm_init,
    .suspend = nfc_pm_suspend,
    .resume = nfc_pm_resume,
};

/* ========== 线程内初始化 ========== */
static void my_nfc_task(void *p1, void *p2, void *p3)
{
    /* ... 其他初始化 ... */

    /* 注册到电源管理模块 */
    my_pm_device_register(MY_PM_DEV_NFC, &nfc_pm_ops);

    /* 恢复设备（首次调用 init） */
    my_pm_device_resume(MY_PM_DEV_NFC);

    /* 延时等待硬件稳定 */
    k_msleep(100);

    /* 初始化完成后挂起，进入低功耗等待 */
    my_pm_device_suspend(MY_PM_DEV_NFC);

    for (;;) {
        /* 主循环 */
    }
}
```

### 4.3 总线管理

| 总线 | 设备 | 管理方式 | 说明 |
|------|------|---------|------|
| I2C22 | NFC | 引用计数 | 支持多设备共享 |
| I2C21 | G-Sensor | 引用计数 | 支持多设备共享 |
| UART | LTE | 直接操作 | 点对点通信，无需计数 |

## 7. 注意事项

1. **线程安全**: 所有 my_pm 接口都是线程安全的
2. **初始化时机**: 必须在各自模块的线程函数内调用 `my_pm_device_register()`
3. **引用计数**: 同一总线多设备场景下，引用计数确保总线正确开关
4. **失败处理**: 回调函数返回错误时，my_pm 会自动回退状态
5. **延时要求**: 硬件唤醒后需要适当延时确保稳定
